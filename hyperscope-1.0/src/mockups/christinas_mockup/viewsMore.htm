Set View More Detail (Beginners):

  1. Set View button is depressed and user has clicked More on the previous Set View command prompt window
  2. The boxes separating the viewspec groups should be a fine line colorless engraved
  3. This adds
    • Time signatures
    • Numbering placement choices (greyed out if Numbering turned off)
    • Labels (node names aka statement names aka anchor names)
    • Level and line choices
    • Content Filtering
  4. This omits (however more adv users can still learn and enter viewspec codes)
    • SIDs/#s: (:IJ) users don’t need at this point and it's too much visually to offer, only devs need
    • All/1st Levels/Lines and combined (:cdstwx) - these are just too much (see example)
    • relative level (:e)
    • Chunking (:ghl)
    • level indenting (:ABQ)
    • Content filter type selector menu – default is simple text search only with quotes implied and optional (regular expression option is deferred)
    • Include on/off (:O/P)
  5. What about
    • Detecting whether SIDS and/or SIGS are supported in a given file, and grey out the options if not
    • Content filter “Match case” and “Match whole words”?
  6. Button details
    1. ?: gives a popup help window describing Set Viewspec command and how to use it
    2. Less: swaps in the Beginners Viewspecs window which includes a Less button in place of the More button
    3. OK: executes the command
    4. Cancel: aborts the command
    5. Reset: clears the viewspecs
