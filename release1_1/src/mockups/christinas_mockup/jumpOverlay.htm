Jump Detail:

  1. Jump button is depressed
  2. Overlays vs. dialog boxes? (See dialog box version of this same prompt window.) I can't remember what we decided about using overlays vs. dialog boxes but the main requirement I can think of is to be able to move it if is blocking your view of what you want to click on
  3. User can click on a node or enter node address or complete URL with optional viewspecs in the typein field
  4. Button details
    1. ?: gives a popup help window describing Jump Item command and how to use it
    2. More: appends the Beginners Viewspecs window and changes the More button to a Less button -- this is the provision for user to learn how to Jump and Set Viewspecs in the same command
    3. OK: executes the jump
    4. Cancel: aborts the command
